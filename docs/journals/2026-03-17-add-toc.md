# Add Table of Contents to README and AGENTS

## What we did

Added auto-generated tables of contents to README.md and AGENTS.md using
doctoc, with a lefthook pre-commit hook to keep them up to date.

### Changes

1. Installed `doctoc@2.3.0` as a dev dependency
2. Added `pnpm toc` script (`doctoc --update-only README.md AGENTS.md`)
3. Inserted doctoc markers and generated initial TOCs in both files
4. Added `lefthook@2.1.4` to `mise.toml` as a project tool
5. Created `lefthook.toml` with a pre-commit hook that runs `pnpm toc`
   when README.md or AGENTS.md are staged (`stage_fixed: true` auto-stages
   the updated files)
6. Restricted the hook glob to `{README,AGENTS}.md` to avoid triggering
   on other markdown files (journals, plans, docs)

- Closes GitHub issue #243

## Decisions

- **lefthook over husky**: Lefthook is a standalone Go binary (no node_modules
  dependency at runtime). Better fit for a Rails project where Node is only
  used for asset bundling. Simpler config (single TOML file, no `prepare`
  script in package.json).

- **lefthook in project mise.toml, not global**: Keeps it reproducible for
  any contributor. Works with both the bare worktree setup and normal clones.

- **No pre-commit hook framework concerns**: The hook installs to
  `.bare/hooks/` in the worktree setup (shared across worktrees). Lefthook
  exits cleanly (exit 0) when no config file is found, so worktrees on
  branches without `lefthook.toml` are unaffected. With a normal clone,
  the hook goes to `.git/hooks/` as usual.

- **Glob restricted to `{README,AGENTS}.md`**: The repo has many markdown
  files in docs/, journals/, plans/. Only README.md and AGENTS.md have
  doctoc markers, so the hook should only trigger for those.
