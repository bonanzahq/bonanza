## Architecture decisions

- **No clockwork gem**: Used Solid Queue's built-in recurring_tasks via config/recurring.yml
- **No extra scheduler container**: The worker process handles both job processing and scheduling
- **Primary DB for queue tables**: No separate queue database - simpler, one less thing to manage
- **deliver_later everywhere**: All mailer calls async via Solid Queue
