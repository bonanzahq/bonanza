## Technical Decisions

### Docker Hub vs GHCR

Switched from GHCR to Docker Hub per Fabian's preference:
- Simpler for external deployments (no GitHub-specific setup needed)
- Standard registry for most Docker installations
- Better discoverability for external collaborators
- Uses username/password authentication instead of GitHub tokens

### Workflow Independence

The docker-build workflow runs independently of the test workflow:
- No `needs: test` dependency
- Tests and builds run in parallel
- Intentional: failed tests don't block Docker builds
- PRs require test pass before merge anyway (GitHub branch protection)
- Keeps workflows decoupled and faster

### paths-ignore on Tags

GitHub Actions automatically ignores `paths-ignore` for tag events, so tag pushes always trigger the workflow. This is correct behavior - we always want to build images for version tags.
